How It Works

  • main.jsx mounts the React app.
  • App.jsx wraps ThemeProvider and AuthProvider.
  • AppRouter.jsx defines public and protected routes.
  • ProtectedRoute.jsx enforces role-based access (student/admin).
  • Auth and theme are stored in localStorage.
  • Data is mock/in-memory from src/data/mockData.js (no backend in this folder).

Frontend Features

  • Role-based auth UI with Login and Signup pages using Student/Admin toggle.
  • Protected routes with role checks and automatic redirects.
  • Session persistence via localStorage.
  • Dark/Light theme toggle with saved preference.
  • Responsive student and admin layouts with desktop and mobile sidebars.
  • Top navigation with search input, notification icon, and profile menu.
  • Student dashboard with welcome banner, stats, activity feed, and trending notes.
  • Notes library with search, subject filter, sorting, and grid/list view switching.
  • Notes upload modal with metadata form and file upload dropzone UI placeholder.
  • Group chat UI with chat list, message thread, message send, and file-message display.
  • Join group modal with join code and copy-to-clipboard support.
  • Leaderboard with rank banner, podium, tier legend, and full ranking table.
  • Feedback module with complaint creation, status filtering, and response tracking.
  • Profile page with editable name/about, user stats, and preferences.
  • Admin dashboard with KPIs, trends, top subjects, and quick panels.
  • Analytics page with user growth, uploads, subject distribution, and top performers.
  • Chat group management for admins with create/delete and join-code copy.
  • Notes moderation with search/filter, flag/unflag, and delete actions.
  • User management with search, warn, and ban/unban controls.
  • Complaint resolution panel with status updates and admin replies.
  • Tailwind-based design system with reusable UI classes.
  • Framer Motion animations for pages, cards, and modals.
  • Lucide icon system across all modules.
  • Custom scrollbars, gradients, and styled utility components.
  • All data is mock/in-memory from src/data/mockData.js (no real backend APIs in this folder).
