# Alembic Database Migrations

This directory contains Alembic database migrations for the ScholarGrid Backend API.

## Overview

Alembic is configured to manage database schema changes for the PostgreSQL database. The configuration automatically reads the database URL from the application's environment variables.

## Configuration

### Environment Variables

The database connection is configured via the `DATABASE_URL` environment variable:

```bash
DATABASE_URL=postgresql+psycopg://user:password@host:port/database
```

### Alembic Configuration Files

- **alembic.ini**: Main Alembic configuration file
- **alembic/env.py**: Environment configuration that loads database URL from app settings
- **alembic/versions/**: Directory containing migration scripts

## Initial Migration

The initial migration (`001_initial_schema.py`) creates all database tables:

### Tables Created

1. **users** - User accounts (students and admins)
2. **notes** - Uploaded educational materials
3. **ratings** - User ratings of notes (1-5 stars)
4. **downloads** - Download tracking records
5. **chat_groups** - Chat group definitions
6. **chat_memberships** - User membership in chat groups
7. **messages** - Chat messages
8. **complaints** - User-submitted issues and feedback
9. **complaint_responses** - Admin responses to complaints
10. **activities** - Platform activity feed events
11. **ai_conversations** - AI chatbot conversation sessions
12. **ai_messages** - Individual messages in AI conversations
13. **ai_usage_tracking** - Daily AI chatbot usage limits

### Schema Features

- **UUID Primary Keys**: All tables use UUID for primary keys
- **Foreign Key Constraints**: Proper relationships with CASCADE delete
- **Check Constraints**: Enum validation for status fields
- **Indexes**: Performance indexes on frequently queried columns
- **Timestamps**: All tables include timezone-aware timestamps

## Usage

### Apply Migrations (Upgrade)

To apply all pending migrations to the database:

```bash
alembic upgrade head
```

### Rollback Migrations (Downgrade)

To rollback all migrations:

```bash
alembic downgrade base
```

To rollback to a specific revision:

```bash
alembic downgrade <revision_id>
```

### View Migration History

To see the current migration status:

```bash
alembic current
```

To see migration history:

```bash
alembic history
```

### Create New Migrations

To create a new migration with autogenerate:

```bash
alembic revision --autogenerate -m "description of changes"
```

To create an empty migration:

```bash
alembic revision -m "description of changes"
```

## Docker Deployment

When using Docker Compose, migrations are automatically applied on startup via the `migrate` service defined in `docker-compose.yml`.

## Troubleshooting

### Connection Issues

If you encounter connection errors:

1. Verify PostgreSQL is running
2. Check DATABASE_URL environment variable
3. Ensure database exists: `createdb scholargrid`
4. Verify credentials are correct

### Migration Conflicts

If you encounter migration conflicts:

1. Check current revision: `alembic current`
2. View history: `alembic history`
3. Resolve conflicts manually or downgrade and re-apply

### Schema Drift

If the database schema doesn't match the models:

1. Generate a new migration: `alembic revision --autogenerate -m "sync schema"`
2. Review the generated migration carefully
3. Apply the migration: `alembic upgrade head`

## Best Practices

1. **Always review autogenerated migrations** before applying them
2. **Test migrations** on a development database first
3. **Backup production data** before running migrations
4. **Use transactions** for data migrations (Alembic does this by default)
5. **Keep migrations small** and focused on single changes
6. **Document complex migrations** with comments in the migration file
